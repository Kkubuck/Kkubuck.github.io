<h2>판다스 데이터프레임의 유용한 메서드</h2>
<ol>
<li><b>데이터프레임 <a href="http://xn--i49a113d.info">객체.info</a>()</b> &rarr; 데이터프레임의 <b>열의 데이터 타입과 누락된 데이터</b>가 있는지 <b>확인</b></li>
<li><b>데이터프레임 객체.describe()</b> &rarr; 열에 대한 간략한 <b>통계</b>를 출력(mean, std, min, max 등등)</li>
</ol>
<p>데이터프레임에서 각 통계 확인 후 <b>특성에 대한 스케일이 다를경우</b> <b>StandardScaler 클래스</b>를 사용해서 특성을 <b>전처리</b> (<b>훈련세트로 학습 후</b> &rarr; 훈련세트와 테스트 세트 전처리)</p>

<p>데이터프레임을 넘파이 배열로 변환 후 훈련세트와 테스트 세트로 나눈다.</p>
<p><b>train_test_split 함수</b>의 매개변수인 <b>test_size</b>는 테스트 세트의 사이즈를 지정한다. 디폴트는 <b>0.25</b>이다.</p>
<p>&nbsp; &nbsp; &nbsp; <b>test_size = 0.2시 입력데이터의 20%를 테스트세트로 변환한다.</b></p>


<h2>결정트리</h2>
<p><b>로지스틱 회귀</b> 모델보다는 <b>결정트리</b> 모델이 <b>설명</b>하기 쉽다.</p>
<p>질문을 하나씩 던져가며 정답을 맞춰가는 과정</p>
<p><figure class="imageblock alignLeft"><img alt="" loading="lazy" src="/assets/img/tistory/25/image-01.png"/></figure>
</p>
<p>사이킷런은 <b>결정트리 알고리즘</b>을 제공한다. <b>DecisionTreeClassifier</b> 클래스로 제공</p>
<p>다른 모델 훈련 클래스와 동일하게 <b>fit()</b> 함수로 훈련한다.</p>
<p><b>트리의 깊이(레벨)</b>을 매개변수로 정해주지 않을 경우 정답을 찾을 때까지 학습한다.</p>
<p>&nbsp; &nbsp; &nbsp; &rarr; <b>훈련테스트</b>에 대해 <b>과대적합</b>된다.</p>

<p><figure class="imageblock alignLeft"><img alt="" loading="lazy" src="/assets/img/tistory/25/image-02.png"/></figure>
</p>
<p>결정 트리의 맨 위의 노트를 <b>루트노드</b>, 맨 아래의 노드를 <b>리프노드</b>라고 한다.</p>
<p><b>plot_tree()</b> 함수에서 트리의 깊이를 제한해서 출력하는 매개변수는 <b>max_depth</b>이고 값을 1로 주면 <b>루트 노드를 제외</b>하고 1개의 노드를 확장해서 출력한다.</p>

<p><b>filled 매개변수</b>를 <b>True</b>로 지정시 클래스에 맞게 <b>노드의 색을 칠할 수 있다</b>.</p>
<p>클래스의 비율이 <b>높아질 경우</b> 색이 <b>진해진다.</b></p>

<p><b>feature_names 매개변수</b>는 노드에 <b>특성의 이름</b>을 전달한다. <b>어떤 기준</b>으로 나뉘는지 확인 가능</p>

<p>부모노드의 기준보다 <b>작거나 같은 값</b>인 샘플들은 <b>왼쪽 노드</b>로 이동, <b>반대</b>는 <b>오른쪽 노드</b>로 이동</p>
<p>왼쪽, 오른쪽으로 나뉜 샘플들은 부모노드의 <b>value</b>에 자세히 나타나있다.</p>


<h3>불순도</h3>
<p><figure class="imageblock alignLeft"><img alt="" loading="lazy" src="/assets/img/tistory/25/image-03.png"/></figure>
</p>
<p>노드 안에 <b>gini</b>는 <b>지니 불순도</b>를 의미한다.</p>
<p><b>DecisionTreeClassifier</b> 클래스의 criterion 매개변수는 노드에서 데이터를 분할할 기준을 정하는 것, 기본값이 <b>gini</b>이다.</p>

<p><b>지니 불순도 = 1 - (음성 클래스 비율^2 + 양성 클래스 비율^2)</b></p>

<p><b>최악</b>의 지니 불순도: 부모노드의 샘플의 비율이 <b>왼쪽 50%, 오른쪽 50%인 경우</b> = <b>0.5가 된다.</b></p>
<p><b>최상</b>의 지니 불순도: 노드에 <b>하나의 클래스</b>만 있는 경우, <b>순수노드</b>라고 한다. = <b>0이 된다.</b></p>

<p>결정 트리 모델은 <b>부모노드와 자식노드의 불순도 차이(정보이득)</b>가 크도록 트리를 성장시킨다.</p>

<p><b>정보이득 = 부모의 불순도 - (왼쪽노드 샘플 수 / 부모의 샘플 수) x 왼쪽 노드 불순도 - (오른쪽노드 샘플 수 / 부모의 샘플 수) x 오른쪽 노드 불순도</b></p>
<p>&nbsp; &nbsp; &nbsp; 노드를 <b>순수하게</b> 나눌수록 정보이득이 <b>커진다.</b></p>

<p>사이킷런에는 엔트로피 불순도도 존재한다. criterion = &lsquo;entropy&rsquo; 지정하여 사용</p>
<p>지니 불순도처럼 제곱이 아니라 밑이 2인 로그(이진 로그)를 사용하여 곱한다.</p>

<p><b>엔트로피 불순도 = 1 - (log2(음성 클래스 비율) x 음성 클래스 비율 - 양성 클래스 비율 x log2(양성 클래스 비율))</b></p>


<h2>가지치기</h2>
<p>제한 없이 끝까지 자라난 트리는 훈련세트에는 적합하지만 테스트세트에서는 맞지 않는다. (과대적합) &rarr; 일반화 되지 않았다.</p>
<p>일반화의 가장 간단한 방법은 <b>학습을 덜 시키는</b> 것이다. &rarr; 트리의 <b>최대 깊이</b>를 지정해서 <b>훈련량</b>을 정하는 것이다. <b>DecisionTreeClassifier</b> 클래스의 <b>max_depth 매개변수</b>에서 지정한다.</p>
<p>&nbsp; &nbsp; &nbsp; &rarr; <b>dt = DecisionTreeClassifier(max_depth=3)</b></p>

<p>이렇게 가지치기 진행시 <b>훈련세트의</b> 성능은 <b>낮아</b>지지만 <b>테스트 세트</b>의 성능은 거의 <b>변함 없다</b>. (과대적합 방지)</p>

<p><b>결정트리</b> 알고리즘은 <b>표준화 전처리</b>를 할 <b>필요가 없다.</b></p>
<p>결정트리는 마지막에 <b>어떤 특성이 분류에 제일 중요</b>하게 작용했는지 나타내는 <b>특성 중요도</b>를 계산해준다.</p>
<p>&nbsp; &nbsp; &nbsp;&nbsp;&rarr; <b>dt.feature_importances_ 로 확인 가능</b></p>
