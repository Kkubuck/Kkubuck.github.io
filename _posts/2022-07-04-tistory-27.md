<p><b>가지런히 정리</b>되어 있는 CSV파일, 엑셀파일등을 <b>정형 데이터</b>라고 부른다.</p>
<p>글과 같은 <b>텍스트 데이터, 사진, 음악</b>등을 <b>비정형 데이터</b>라고 부른다.</p>
<p><b>정형 데이터</b>를 다루는데 <b>가장 뛰어난 성과</b>를 내는 알고리즘이 <b>앙상블 학습</b>이다.</p>


<h2>랜덤 포레스트</h2>
<p>앙상블 학습의 대표 주자 중 하나로 <b>안정적인 성능</b>을 낸다.</p>
<p>결정트리를 <b>랜덤</b>하게 만들어 <b>결정트리의 숲</b>을 만든다.</p>
<p><b>각 결정 트리의 예측</b>을 사용해 <b>최종 예측</b>을 만든다.</p>
<p><figure class="imageblock alignLeft"><img alt="" loading="lazy" src="/assets/img/tistory/27/image-01.png"/></figure>
</p>
<p>입력한 훈련 데이터에서 <b>랜덤</b>하게 샘플을 추출하여 <b>훈련 데이터</b>를 만든다. &rarr; 샘플의 <b>중복</b>을 허용한다.</p>
<p>이렇게 만들어진 샘플을 <b>부트스트랩 샘플</b>이라고 부른다.</p>
<p><figure class="imageblock alignLeft"><img alt="" loading="lazy" src="/assets/img/tistory/27/image-02.png"/></figure>
</p>
<p>&rarr; 기본적으로 <b>부트스트랩 샘플의 크기</b>는 <b>훈련세트의 크기</b>와 <b>같게</b> 만든다.</p>

<p>각 노드를 분할할 때 <b>전체 특성 중에서 일부 특성을 무작위</b>로 고른 다음 <b>최선의 분할을 찾는다.</b></p>

<p>분류 모델인 <b>RandomForestClassifier</b>은 기본적으로 <b>전체 특성 개수의 제곱근만큼의 특성</b>을 선택</p>
<p>&rarr; <b>4개</b>의 특성이 있는경우 노드마다 <b>2개</b>를 랜덤으로 선택하여 사용</p>

<p>회귀 모델인 <b>RandomForestRegressor</b>은 <b>전체 특성</b>을 사용한다.</p>

<p>사이킷런의 랜덤 포레스트는 <b>기본적으로 100개</b>의 결정트리를 이런 방식으로 훈련한다.</p>

<p>랜덤 포레스트는 랜덤하게 선택했으므로 <b>훈련세트에 과대적합</b>되는것을 <b>방지</b>하고 테스트 세트에서 <b>안정적인 성능</b>을 얻을 수 있다.<b>(일반화 성능을 높임)</b></p>

<pre class="python"><code>scores = cross_validate(rf, train_input, train_target, return_train_score=True, n_jobs=-1)
print(np.mean(scores['train_score']), np.mean(scores['test_score']))</code></pre>
<p><b>cross_validate 함수</b>의 매개변수인 <b>return_train_score</b>은 <b>기본값이 False</b>이므로 평균을 출력하고 싶을 경우 <b>True값으로 지정</b>해줘야 한다.</p>

<p><b>랜덤 포레스트</b>는 <b>결정 트리의 앙상블</b>이기때문에 <b>DecisionTreeClassifier</b>가 제공하는 <b>중요한 매개변수</b>를 모두 제공합니다.</p>
<p><b>&rarr; criterion, max_depth, max_features, min_samples_split, min_impurity_decrease, min_samples_leaf 제공</b></p>

<p><b>RandomForestClassifier</b> 클래스는 <b>자체적</b>으로 <b>모델을 평가하는 점수</b>를 얻을 수 있다.</p>

<p><b>부트스트랩 샘플</b>에 <b>포함되지 않고</b> 남는 샘플 &rarr; <b>OOB(out of bag) 샘플</b>이라고 한다.</p>
<p>이 <b>OOB샘플</b>을 사용하여 <b>부트스트랩 샘플로 훈련한</b> <b>결정 트리</b>를 <b>평가</b>할 수 있다.</p>
<p><b>OOB세트</b>가 <b>검증 세트의 역할</b>을 한다고 볼 수 있다.</p>
<p>점수를 얻기 위해서는 <b>RandomForestClassifier</b> 클래스의 <b>oob_score</b> 매개변수를 <b>True</b>로 지정해야 함.(기본값이 <b>False</b>)</p>
<pre class="python"><code>rf = RandomForestClassifier(oob_score=True, n_jobs=-1, random_state=42)
rf.fit(train_input, train_target)
print(rf.oob_score_)</code></pre>
<p><b>OOB점수</b>를 사용하면 <b>교차 검증</b>을 <b>대신</b>할 수 있어서 결과적으로 훈련 세트에 <b>더 많은 샘플을 사용</b>할 수 있다.</p>


<h2>엑스트라 트리</h2>
<p><b>랜덤포레스트</b>와 비슷하게 동작, 기본적으로 <b>100개의 결정트리</b>를 훈련</p>
<p><b>랜덤포레스트</b>와 동일하게 <b>결정트리가 제공</b>하는 <b>매개변수</b>들을 지원</p>

<p><b>가장 좋은 분할</b>을 찾는 것이 <b>아니라</b> 전체 특성 중 <b>일부 특성을 랜덤</b>하게 선택하여 <b>노드를 분할</b>한다.</p>
<p>&rarr; splitter 매개변수를 random으로 지정</p>

<p><b>랜덤</b>으로 노드를 분할하기 때문에 <b>빠른 계산 속도</b>가 특징이다.</p>
<p><b>하나의 결정 트리</b>에서 특성 <b>무작위로 분할</b> &rarr; <b>성능 감소</b></p>

<p><b>많은 트리를 앙상블</b>하기 때문에 <b>과대적합을 막고</b> <b>검증 세트의 점수를 높이는 효과</b>가 있다.</p>
<p>사이킷런에서 제공하는 <b>엑스트라 트리</b>는 <b>ExtraTreesClassifier 클래스</b></p>
<pre class="python"><code>from sklearn.ensemble import ExtraTreesClassifier

et = ExtraTreesClassifier(n_jobs=-1, random_state=42)
scores = cross_validate(et, train_input, train_target, return_train_score=True,
												n_jobs=-1)
print(np.mean(scores['train_score']), np.mean(score['test_score']))</code></pre>
<p><b>엑스트라 트리</b>는 <b>부트스트랩 샘플</b>을 <b>사용하지 않는다.</b></p>
<p>&rarr; <b>결정트리</b>를 만들 때 <b>전체 훈련 세트</b>를 사용한다.</p>

<p>엑스트라 트리의 <b>회귀 버전</b>은 <b>ExtraTreesRegressor</b></p>


<h2>그레이디언트 부스팅</h2>
<p><b>깊이가 얕은 결정 트리</b>를 사용하여 이진 트리의 <b>오차를 보완</b>하는 방식으로 <b>앙상블</b>하는 방법</p>
<p>사이킷런의 <b>GradientBoostingClassifier 클래스</b>로 제공</p>
<p>&rarr; 기본적으로 <b>깊이가 3인 결정 트리 100개</b> 사용</p>

<p>깊이가 <b>얕기</b> 때문에 <b>과대적합에 강력</b>하고 <b>높은 일반화 성능</b>을 기대할 수 있다.</p>
<p><b>경사 하강법</b>을 사용하여 트리를 앙상블에 추가한다.</p>
<p>&rarr; 결정 트리를 <b>계속 추가</b>하면서 <b>가장 낮은 곳을 찾아 이동</b>, <b>학습률 매개변수</b>로 <b>속도</b>를 조정</p>

<p><b>분류</b>에서는 <b>로지스틱 손실 함수</b>를 사용하고 <b>회귀</b>에서는 <b>평균 제곱 오차 함수</b> 사용</p>
<pre class="python"><code>from sklearn.ensemble import GradientBoostingClassifier

gb = GradientBoostingClassifier(random_State=42)
scores = cross_validate(gb, train_input, train_target, return_train_scores=True, n_jobs=-1)

print(np.mean(scores['train_score']), np.mean(scores['test_score']))</code></pre>
<p><b>학습률을 증가</b>시키고 <b>트리의 개수를 증가</b>시키면 <b>성능 향상 가능</b></p>

<p><b>트리 훈련에 사용할 훈련 세트의 비율</b>을 정하는 <b>subsample</b>의 <b>기본값은 1</b>이다.</p>
<p><b>1일</b> 경우 &rarr; <b>전체 훈련 세트</b> 사용</p>
<p>1보다 <b>작을</b> 경우 &rarr; <b>훈련 세트의 일부</b> 사용</p>

<p>그레이디언트 부스팅의 <b>회귀</b>버전은 <b>GradientBoostingRegressor 클래스</b></p>


<h2>히스토그램 기반 그레이디언트 부스팅</h2>
<p><b>정형데이터</b>를 다루는 머신러닝 알고리즘 중에 가장 <b>인기가 높은</b> 알고리즘</p>
<ol>
<li>입력 특성의 <b>256개의 구간</b>으로 나눈다. &rarr; <b>최적의 분할</b>을 <b>매우 빠르게</b> 찾을 수 있다.</li>
<li><b>256개구간</b> 중 <b>하나</b>를 떼어 놓고 <b>누락된 값</b>을 위해서 사용 &rarr; <b>누락된 특성</b> <b>전처리</b>가 필요 없다.</li>
</ol>
<p>사이킷런의 <b>HistGradientBoostingClassifier 클래스</b>로 제공</p>
<p><b>HistGradientBoostingClassifier</b>에서는 <b>트리의 개수를 지정</b>하는데 <b>n_estimators</b> 대신 <b>부스팅 반복 횟수</b>를 지정하는 <b>max_iter</b>을 사용한다.</p>
<pre class="python"><code>from sklearn.experimental import enable_hist_gradient_boosting
from sklearn.ensemble import HistGradientBoostingClassifier

hgb = HistGradientBoostingClassifier(random_state=42)
scores = cross_validate(hgb, train_input, train_target, return_train_score=True)

print(np.mean(scores['train_score']), np.mean(scores['test_score']))</code></pre>
<p><b>HistGradientBoosting</b>의 <b>특성 중요도</b>를 계산하기 위해 <b>permutation_importance()</b> 함수 사용</p>
<p>&rarr; <b>하나씩 랜덤하게 섞어서</b> 모델의 <b>성능이 변화</b>하는지 관찰하여 <b>어떤 특성이 중요한지</b> 계산</p>

<p>훈련세트, 테스트세트, 사이킷런에서 제공하는 추정기 모델에도 사용 가능</p>

<p>매개변수인 <b>n_repeats</b>는 <b>랜덤하게 섞을 횟수</b>를 지정한다. <b>기본값은 5</b></p>
<p><b>특성중요도, 평균, 표준편차</b>를 <b>반환</b>한다.(permutation_importance())</p>
<pre class="python"><code>from sklearn.inspection import permutation_importance

hgb.fit(train_input, train_target)
result = permutation_importance(hgb, train_input, train_target, n_repeats=10, random_state=42, n_jobs=-1)

print(result.importances_mean)</code></pre>
<p><b>히스토그램 기반 그레이디언트 부스팅</b>의 <b>회귀</b> 버전은 <b>HistGradientBoostingRegressor </b>클래스</p>
