<h2>검증세트</h2>
<p><b>테스트세트</b>를 <b>사용하지 않고</b> 모델이 과소적합인지 과대적합인지 판단하기 위해 <b>훈련세트</b>를 또 <b>나눠서</b> <b>검증세트</b>를 만든다.</p>
<p><figure class="imageblock alignLeft"><img alt="" loading="lazy" src="/assets/img/tistory/26/image-01.png"/></figure>
</p>
<p><b>훈련세트</b>에서 모델을 <b>훈련</b>하고 <b>검증 세트</b>로 모델을 <b>평가</b>한다.</p>
<pre class="python"><code>from sklearn.model_selection import train_test_split

#데이터와 타깃을 훈련세트와 테스트세트로 분할
train_input, test_input, train_target, test_target = train_test_split(data, 
target, test_size = 0.2)

#훈련세트를 훈련세트와 검증세트로 분할(sub: 훈련세트, val: 검증세트)
sub_input, val_input, sub_target, val_target = train_test_split(train_input, 
train_target, test_size = 0.2)</code></pre>
<p>위 코드에서 분할한 <b>sub_input, sub_target(훈련세트)</b>로 모델을 <b>훈련</b>한다.</p>
<p><b>테스트세트 대신</b> <b>val_input, val_target(검증세트)</b>로 모델을 <b>평가</b>한다.</p>


<h2>교차 검증</h2>
<p>검증세트를 너무 조금 떼어 놓을 경우 검증 점수가 <b>불안정하다</b>. &rarr; <b>교차 검증</b>을 이용해 <b>안정적인 점수</b>를 얻고 <b>훈련에 더 많은 데이터를 사용</b>한다.</p>
<p>교차 검증은 검증 세트를 떼어 평가하는 과정을 <b>여러번 반복하고</b> 이 점수를 <b>평균</b>내서 <b>최종 검증 점수</b>를 얻는다.</p>
<p><figure class="imageblock alignLeft"><img alt="" loading="lazy" src="/assets/img/tistory/26/image-02.png"/></figure>
</p>
<p>훈련세트를 <b>k부분으로 나눠서</b> 교차 검증을 수행하는 것을 <b>k-겹 교차 검증</b>이라고 한다.</p>
<p><b>폴드</b>는 훈련세트를 <b>몇부분으로 나누냐</b>에 따라 달라진다.</p>
<p>보통 <b>5, 10-폴드 교차 검증</b>을 많이 사용한다. &rarr; 데이터의 <b>80~90%를</b> 훈련에 사용 가능</p>
<p>사이킷런에는 <b>cross_validate()</b>라는 <b>교차검증 함수</b>가 존재한다.</p>
<p>평가할 <b>모델 객체를 첫번째 매개변수</b>로 전달하고 <b>훈련세트를 두번째 매개변수</b>로 전달한다.</p>
<pre class="python"><code>from sklearn.model_selection import cross_validate

scores = cross_validate(dt, train_input, train_target)</code></pre>
<p>&rarr; <b>fit_time(훈련시간), score_time(검증시간), test_score(최종점수)</b> 키를 가진 <b>딕셔너리</b>를 반환한다.</p>

<p>교차검증을 수행하면 입력한 모델에서 얻을 수 있는 최상의 검증 점수를 가능 가능</p>

<p>train_test_split에서 훈련세트를 섞지 않고 cross_validate에서 훈련세트를 섞으려면 분할기를 지정해야 한다.</p>
<p>&rarr; <b>사이킷런의 분할기</b>는 교차 검증에서 <b>폴드를 어떻게 나눌지</b> 결정해준다.</p>

<p>cross_validate 함수는 기본적으로 <b>분류</b> 모델일 경우 <b>StratifiedKFold</b> 분할기를 사용한다.</p>
<p><b>회귀</b> 모델일 경우 <b>KFold</b> 분할기를 사용한다.</p>

<p><b>StratifiedKFold</b> 분할기의 매개변수중 <b>n_splits</b>는 <b>교차검증의 폴드수를 </b>정합니다.</p>
<p><b>shuffle 매개변수</b>를 True로 지정시 <b>훈련세트</b>를 <b>섞는다</b>.</p>


<h2>하이퍼파라미터 튜닝</h2>
<p>머신러닝 <b>모델이 학습하는 파라미터</b>를 <b>모델 파라미터</b>라고 부른다.</p>
<p>모델이 학습 불가하여 <b>사용자가 직접 지정</b>하는 파라미터를 <b>하이퍼파라미터</b>라고 부른다.</p>
<p>&rarr; 머신러닝 라이브러리 사용시 <b>클래스나 메서드의 매개변수</b>로 표현된다.</p>
<p><b>튜닝하는 작업</b></p>
<ol>
<li>라이브러리가 제공하는 <b>기본값을 사용하여 모델 훈련</b></li>
<li>검증세트나 교차검증을 통해서 <b>매개변수를 조금씩 변경</b></li>
<li>여러개의 <b>매개변수의 최적값을 동시에 바꿔가며</b> 찾아야 한다 &rarr; 한개 찾고 다음꺼 찾으면 안됨</li>
</ol>
<p>매개변수가 많아지면 문제는 더욱 <b>복잡</b>해진다.</p>
<p>&rarr; 사이킷런에서 제공하는 <b>그리드 서치</b>를 사용한다.</p>

<p><b>GridSearchCV 클래스</b>는 <b>하이퍼파라미터 탐색과 교차 검증을 한번에 수행</b>한다.</p>
<p>별도로 교차검증을 할 필요가 없다. <b>(cross_validate 함수 필요없음)</b></p>

<p>탐색할 매개변수와 탐색할 값의 리스트를 <b>params 딕셔너리로</b> 만든다.</p>
<p><b>&rarr; params = {&rsquo;min_impurity_decrease&rsquo;: [0.0001, 0.0002, 0.0003, 0.0005]}</b></p>

<p>결정트리 클래스의 객체를 <b>생성하자마자</b> 그리드서치 매개변수로 <b>바로 전달하는것이 가능</b>하다.</p>
<p><b>&rarr; gs = GridSearchCV(DecisionTreeClassifier(random=42), params, n_jobs=-1)</b></p>
<p>이 메서드 호출시 그리드 서치 객체는 결정트리모델 <b>min_impurity_decrease 값</b>을 바꿔가며 총 5번 실행한다. (GridSearchCV의 <b>cv 매개변수 기본값은 5</b>)</p>

<p><b>훈련이 끝나면</b> <b>25개의 모델중</b>에서 <b>검증 점수가 가장 높은 모델</b>의 매개변수 조합으로 전체 훈련 세트에서 <b>자동으로 다시 모델을 훈련</b>한다. &rarr; 훈련한 모델은 <b>best_estimator</b>_ 속성에 저장되어 있다.</p>

<p>이 모델을 <b>일반 결정트리</b>처럼 사용이 가능하다.</p>
<p><b>그리드 서치</b>로 찾은 최적의 매개변수는 <b>best_params</b>_속성에 저장</p>
<p>각 매개변수에서 수행한 <b>교차 검증의 평균 점수</b>는 <b>cv_result 속성의 &lsquo;mean_test_score&rsquo;키</b>에 저장</p>
<p>&rarr; <b>gs.cv_result[&lsquo;mean_test_score&rsquo;]</b></p>

<p><b>params</b>에 <b>더욱 복잡한 매개변수 조합</b>을 넣을 수도 있다.</p>
<pre class="python"><code>params = {'min_impurity_decrease': np.arange(0.0001, 0.001, 0.0001), #불순도 감소 최소량: 9개

          'max_depth': range(5, 20, 1), #트리의 깊이: 15개

          'min_samples_split': range(2, 100, 10) #노드를 나누기 위한 최소 샘플 수: 10개 }</code></pre>
<p><b>np.arange와 range 차이</b>는 <b>range 함수</b>는 <b>정수</b>만 가능하나 <b>np.arange 함수</b>는 <b>실수</b>도 가능하다</p>
<p>이 params 매개변수로 수행할 <b>교차 검증 횟수</b>는 <b>9 x 15 x 10 = 1,350개</b>이다.</p>
<p>기본 <b>5-폴드 교차 검증</b>을 수행하므로 <b>만들어지는 모델의 수</b>는 <b>1350 x 5 = 6750개</b>가 됩니다.</p>

<p>그리드 서치 객체로 매개변수 <b>n_jobs = -1</b>로 지정시 <b>가능한 모든 코어를 사용</b>해서 <b>계산</b></p>


<h2>랜덤서치</h2>
<p>매개변수의 값이 수치일 경우 범위나 간격을 <b>미리 정하기 어려울 때</b>는 <b>랜덤 서치</b>를 사용한다.</p>
<p>매개변수 값의 목록을 전달하는 것이 아니라 매개변수를 샘플링 할 수 있는 <b>확률 분포 객체</b>를 전달한다.</p>

<p>사이파이에서 <b>2개의 확률 클래스</b>를 임포트</p>
<p><b>&rarr; from scipy.stats import uniform, randint</b></p>
<p><b>균등분포</b>에서 샘플링한다. <b>randint에서는 정수, uniform에서는 실수</b>를 뽑는다.</p>
<p><b>rgen = randint(0, 10)</b> &rarr; 0에서 9까지의 <b>랜덤한 정수</b></p>
<p><b>rgen.rvs(10)</b> &rarr; 10개의 랜덤한 변수를 <b>샘플링</b></p>
<p>uniform도 똑같이 수행하면 된다.</p>

<p><b>min_samples_leaf 매개변수</b>는 노드가 분할하여 만들어질 <b>자식 노드의 샘플수가 이 값보다 작을 경우 분할하지 않는다.</b></p>
<pre class="python"><code>params = {'min_impurity_decrease': uniform(0.0001, 0.001)

          'max_depth': randint(20, 50)

          'min_samples_split': randint(2, 25)
									
		  'min_samples_leaf': randint(1, 25)              
		  }

from sklearn.model_selection import RandomizedSearchCV

#n_iter는 샘플링 횟수 지정(prarms에 있는 랜덤분포에서의 샘플링 횟수)
gs = RandomizedSearchCV(DecisionTreeClassifier(random_state=42), params, n_iter=100, 
n_jobs=-1, random_state=42)

gs.fit(train_input, train_target)</code></pre>
