# Kkubuck Research Blog — Full Redesign Prompt

당신은 세계적 수준의 디지털 제품 스튜디오에 소속된 시니어 아트 디렉터, UX 엔지니어, 인터랙션 디자이너, 프론트엔드 아키텍트로 구성된 팀이다. 기존 Jekyll 기반 GitHub Pages 블로그의 공개 콘텐츠와 URL 호환성을 유지하면서, 사이트 전체를 논문 리뷰와 개인 연구 기록에 특화된 프리미엄 리서치 퍼블리싱 플랫폼으로 전면 재설계하고 구현하라.

## 1. 핵심 인상

- 첫 화면부터 “고도로 전문화된 연구자와 숙련된 프론트엔드 팀이 공동 제작한 사이트”라는 인상을 준다.
- 전형적인 블로그 템플릿이나 단순 카드 나열을 피하고, 편집 디자인의 명확한 정보 위계와 실험적인 인터랙션을 결합한다.
- 유리 재질은 장식이 아니라 깊이, 계층, 맥락을 표현하는 기능적 레이어로 사용한다. 과도한 blur, 낮은 대비, 읽기 어려운 투명도는 금지한다.
- 연구 노트, 논문 메타데이터, 벤치마크, 코드, 프로젝트가 하나의 지식 시스템처럼 연결되어 보여야 한다.

## 2. 시각 언어

- 배경: 짙은 잉크색과 차가운 청록·보라 계열의 저채도 mesh gradient, 미세한 노이즈, 은은한 광원.
- 표면: 높은 대비의 반투명 유리 패널, 얇은 스펙트럴 보더, 미세한 내부 하이라이트, 깊이감 있는 그림자.
- 타이포그래피: 한국어 가독성이 높은 산세리프를 본문에, 연구 저널의 인상을 주는 세리프를 핵심 헤드라인과 인용에 제한적으로 사용한다.
- 정보 표현: 논문 번호, venue, 연도, 태그, 읽기 시간, 출처를 정교한 마이크로 타이포그래피로 구성한다.
- 모션: 느리고 정확한 easing, pointer 반응형 parallax, orbital rotation, magnetic hover, scroll reveal. 모든 모션은 `prefers-reduced-motion`을 존중한다.

## 3. 대표 인터랙션

- 일반적인 수평 메뉴 대신 접근 가능한 “Research Orbit” 원형 탐색기를 제공한다.
  - 중앙 버튼을 누르면 홈, 글, 논문, 프로젝트, CV, 소개가 원호 또는 원형으로 확장된다.
  - 키보드 탐색, ESC 닫기, 바깥 영역 클릭 닫기, 현재 페이지 표시를 지원한다.
  - 작은 화면에서는 동일한 기능을 읽기 쉬운 glass sheet 형태로 전환한다.
- `Ctrl/Cmd + K`로 여는 Research Command Palette를 제공한다.
  - 전체 포스트와 주요 페이지를 제목, 요약, 태그, venue로 즉시 검색한다.
  - 화살표 키, Enter, ESC 조작을 지원한다.
- 홈 히어로에는 연구 주제가 궤도를 도는 인터랙티브 “Knowledge Observatory”를 둔다.
- 논문 아카이브는 검색, venue, 연도, 태그 필터와 결과 수 갱신을 지원한다.
- 글 상세에는 읽기 진행도, 자동 생성 목차, heading anchor, 공유/링크 복사, 원문/PDF 링크, 이전·다음 글 탐색을 제공한다.

## 4. 페이지별 요구사항

### Home
- 강한 editorial headline, 연구 범위, 최신 업데이트 상태, 핵심 통계를 배치한다.
- 최신 논문 리뷰를 가장 중요한 콘텐츠로 보여주고, 일반 글과 프로젝트를 분리해 보여준다.
- 최근 포스트는 번호가 붙은 research index 또는 journal issue처럼 구성한다.
- 전체 글, 논문, 프로젝트, 태그, CV로 가는 명확한 경로를 제공한다.

### Papers
- 논문 리뷰 전용 아카이브로 설계한다.
- 검색/필터, venue badge, 연도, 태그, 요약, 원문 링크 정보를 제공한다.
- 모바일에서 필터가 접히고 목록이 한 열로 자연스럽게 흐른다.

### Blog / Projects / Tags
- 블로그는 시간순 연구 로그, 프로젝트는 case study, 태그는 지식 지도처럼 보이게 한다.
- 태그 페이지에는 시각적 constellation과 접근 가능한 전체 목록을 함께 제공한다.

### Post
- 긴 한국어 글의 가독성을 최우선으로 한다.
- 본문 폭, 행간, 표, 코드, 인용문, 이미지, 캡션, iframe을 정교하게 처리한다.
- 논문 포스트에는 venue, source, PDF, tags를 별도 metadata panel로 보여준다.
- 데스크톱에서 sticky TOC, 모바일에서 접이식 TOC를 제공한다.

### CV / About / 404
- CV는 academic dossier처럼 정돈하되 사이트 시각 언어와 일관되게 한다.
- About은 연구 관심사와 블로그 운영 원칙을 명료하게 설명한다.
- 404도 동일한 원형 인터랙션과 브랜드 언어를 유지한다.

## 5. 기술 및 품질 조건

- Jekyll과 GitHub Pages 배포 흐름을 유지한다.
- 기존 포스트 콘텐츠와 permalink를 변경하지 않는다.
- 외부 JavaScript 프레임워크 없이 HTML, Liquid, SCSS/CSS, vanilla JavaScript로 구현한다.
- JavaScript가 꺼져도 주요 콘텐츠와 탐색 링크가 보이는 progressive enhancement 구조로 만든다.
- 반응형, 키보드 접근성, 명확한 focus state, 충분한 색 대비, `aria-*`, reduced motion을 적용한다.
- CLS를 줄이고, 이미지 lazy loading, defer script, 최소한의 외부 요청으로 성능을 관리한다.
- dark/light theme를 제공하고 사용자 선택을 `localStorage`에 저장한다.
- 빌드 경고, Liquid 오류, 깨진 내부 링크, 누락 자산, JavaScript 문법 오류를 검증한다.
- 최종 결과물은 저장소 루트에 그대로 업로드하면 GitHub Pages로 배포 가능해야 한다.

## 6. 금지 사항

- 흔한 SaaS 랜딩 페이지 복제, 과도한 네온, 읽기 어려운 투명 패널, 자동 재생 영상, 무거운 WebGL 의존성.
- 마우스 사용자만 쓸 수 있는 인터랙션.
- 의미 없는 카드 반복, 모든 요소에 blur 적용, 모바일에서 사라지는 핵심 기능.
- 기존 글 내용의 임의 삭제 또는 사실 관계 변경.

## 7. 최종 산출물

- 전면 개편된 Jekyll 소스 전체.
- 디자인 토큰과 컴포넌트 스타일.
- 원형 탐색기, 검색 팔레트, 필터, 목차, 테마 전환을 포함한 JavaScript.
- 수정된 README와 구현 프롬프트 문서.
- 로컬 빌드 및 정적 검증을 통과한 배포용 ZIP.
