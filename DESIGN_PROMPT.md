# Kkubuck 연구 블로그 디자인·구현 프롬프트

기존 Jekyll/GitHub Pages 저장소의 모든 콘텐츠, URL, front matter, 이미지, 데이터 파일을 보존하면서 UI를 전면 개편한다.

## 목표

- 논문 리뷰와 연구 기록을 다루는 **전문 연구 아카이브**처럼 보여야 한다.
- 과장된 대형 타이포, 복잡한 장식, 거대한 전체 화면 메뉴를 피한다.
- 짙은 네이비 배경과 블루·시안 계열 포인트, 반투명 유리 표면을 사용한다.
- 정보 밀도는 높되 여백과 계층이 분명해야 한다.
- 메뉴와 공통 인터페이스 문구는 한국어로 통일한다. 논문 제목, 학회명, 기술 용어는 원문을 유지한다.

## 시각 시스템

- 배경: `#050a15`에 가까운 짙은 네이비, 매우 약한 블루 라디얼 글로우와 미세 그리드.
- 표면: 투명도 55~75%, 16~22px blur, 얇은 블루 회색 테두리.
- 강조색: `#7bb5ff`, `#71d7ff`; 녹색은 사용하지 않는다.
- 제목 최대 크기: 홈 약 55px, 목록 약 48px, 글 본문 제목 약 46px. 모바일에서는 36~40px 수준으로 축소한다.
- 카드 모서리: 14~22px. 과도한 네온과 그림자는 피한다.
- 본문 폭: 약 760px. 행간 1.75~1.85, 읽기 쉬운 계층을 유지한다.

## 정보 구조

- 상단: 작고 고정된 글라스 헤더, 브랜드, 한국어 데스크톱 메뉴, 검색, 테마 전환, 모바일 메뉴.
- 홈: 절제된 소개, 통계, 작은 인터랙티브 연구 주제 오비트, 최근 논문 리뷰, 최근 연구 노트.
- 논문 리뷰: 검색 + 학회/연도/주제 필터, 압축된 카드형 목록.
- 연구 노트: 날짜와 유형이 명확한 시간순 목록.
- 프로젝트: 2열 카드, 문제·구현 기록 중심.
- 태그: 거대한 별자리 장식 대신 작은 태그 클라우드와 검색 가능한 그룹.
- 글 본문: 중앙 정렬 제목, 문서 링크 도구막대, 데스크톱 고정 목차, 760px 읽기 영역, 읽기 진행률.
- CV: 프로필, 연락처, 학력, 논문, 프로젝트, 수상, 교육 활동을 정돈된 데이터 문서 형태로 표시한다.

## 인터랙션

- `Cmd/Ctrl + K` 검색 팔레트.
- `/` 키로 아카이브 검색창 포커스.
- 필터 결과 수 즉시 갱신.
- 홈 오비트의 느린 회전과 포인터 기반 3D 반응.
- 카드에는 2~4px 이내의 미세한 이동과 테두리 강조만 적용.
- 글 목차 활성 상태, 읽기 진행선/원형 진행 표시, 링크 복사와 공유.
- `prefers-reduced-motion`에서 모든 불필요한 모션을 제거한다.

## 구현 조건

- 순수 Jekyll/Liquid/CSS/Vanilla JavaScript로 동작한다.
- GitHub Pages Actions 배포 구성을 유지한다.
- 기존 글과 정적 자산 경로를 변경하지 않는다.
- 키보드 탐색, 포커스 트랩, 명확한 `aria-label`, 충분한 대비를 제공한다.
- 320px 모바일부터 대형 데스크톱까지 가로 스크롤이 없어야 한다.
- 빌드, JavaScript 문법, 내부 링크, 정적 자산 경로, 주요 화면 렌더링을 검증한다.
